# Layout Reference

This site uses mature technical publishing patterns without copying any single brand.

## Editorial Technology Magazines

- Strong masthead rhythm: large title, brief deck, clear sections.
- Story lists favor chronology, hierarchy, and metadata over decorative cards.
- Article pages keep the reading column narrow and let headings, quotes, and code create the pace.

## Developer And Product Blogs

- The home page should show what the author is currently thinking about, not just a feed.
- Project sections benefit from compact technical status, stack tags, and clear next links.
- Navigation stays shallow: Home, Blog, Projects, About.

## Documentation Sites

- Typography must support scanning: short headings, stable tag styles, predictable article metadata.
- Code blocks need dark contrast, horizontal scrolling, and readable monospace.
- Every content type should be editable from a small number of obvious files.

## Constraints For This Project

- WIRED-inspired editorial structure is the primary visual model.
- Linear-inspired surfaces are limited to engineering cards, code panels, metadata, and hover states.
- No marketing hero template, no heavy motion, no decorative gradients, no borrowed brand assets.
