一台小 VPS 在正常运行时看起来很简单，直到第一次故障发生。那时，每个没有记录的决定都会变贵：防火墙规则、SSH 入口、DNS 记录、反向代理配置、证书续期、进程守护。

更实用的写法，是按照排障时真正需要的顺序记录：

- 如何连接服务器
- 哪些服务应该正在运行
- 日志在哪里
- 证书如何续期
- DNS 记录应该是什么
- 最近改过什么

## 最小检查

```bash
systemctl status nginx
nginx -t
curl -I https://example.com
```

运维笔记不一定要长，但它必须在系统已经出问题的时候还能被快速理解。
